   1. <span data-ttu-id="2d82f-114">これが追跡クエリの場合、EF では、データがコンテキスト インスタンスの変更追跡に既にあるエンティティを表しているかどうかをチェックします。</span><span class="sxs-lookup"><span data-stu-id="2d82f-114">If this is a tracking query, EF checks if the data represents an entity already in the change tracker for the context instance</span></span>

<span data-ttu-id="2d82f-120"><sup>(1)</sup> 既に返されたエンティティの追跡を継続するために、追跡なしクエリでは弱参照が使用されます。</span><span class="sxs-lookup"><span data-stu-id="2d82f-120"><sup>(1)</sup> No-tracking queries use weak references to keep track of entities that have already been returned.</span></span> <span data-ttu-id="2d82f-121">同じ ID の前の結果が範囲外になっている場合は、ガベージ コレクションが実行され、新しいエンティティ インスタンスを取得できます。</span><span class="sxs-lookup"><span data-stu-id="2d82f-121">If a previous result with the same identity goes out of scope, and garbage collection runs, you may get a new entity instance.</span></span>
